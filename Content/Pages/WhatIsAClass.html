What is A Class?

In this article, we will discuss the properties of a Class.
This article assumes that you have already Built A MonoGame Project.
Let's get started.


Classes Defined

A class is a collection of variables and functions that you define, like a blueprint.
The variables and functions of a class are called members of the class.
When you define a function in a class, that function can be referred to as a method.

Objects are instances of a class.
So when you call a classes function, you are accessing an object's method.
This is done using the dot (.) operator.

Methods That Take Parameters

Note the method defined below, Update().

You can see that Update has an object in it's parentheses called GameTime gameTime.
GameTime defines the type. gameTime is the name of the object.
We could change the name of the object to something different, for example: GameTime updateTime.

An object in a function or method's parentheses is known as a parameter.
We would describe the Update method as taking one parameter, an object of the GameTime type.
This parameter would be available inside Update's scope, which is defined by curly brackets {}.





Scope Defined

Note the curly brackets {} below in Update().

These curly brackets define the scope of the Update() method.
Parameters that are passed to a method become available inside a method's scope.
However, a method's parameters cannot be used outside of the method's scope.

This is true for members of a class too.
Members defined inside a classes scope cannot be used outside of the class.
A classes members are accessed using the dot (.) operator.
