What is a Codebase?

The term codebase refers to the collection of source code that is used to build a game.
This includes the game's classes, and any other libraries or dependencies the game may have.
In this article we will only be discussing the example game's classes.





Inspecting RescueDog's Classes

RescueDog is composed of a small number of classes that interact with each other to make the game function.
When one class interacts with another class, the two classes are said to be 'coupled'.
This means the two classes are interdependent, or dependent on each other.
Modifying one class may require that the other class be modified as well.

As a game grows, you will likely add more classes and more connections between these classes.
Depending upon how you architect or structure these connections, your codebase may become overly complex.
Changing one class may cause a ripple effect, forcing you to change many other classes.
It's important that you design these connections well, so changes in one class don't ripple.

RescueDog's classes have many connections between them, and can be defined as a tightly coupled codebase.
However, each class has a specific responsibility and that responsbility is limited to only that class.
You should also know that almost all class members are defined with the Public access modifier.
Therefore, RescueDog's codebase rarely implements the OOP concept of Encapsulation.

Why was the codebase created this way? To keep it simple to understand and easy to modify.
If an event based system were used instead of coupled classes, it would be necessary to explain events and delegates.
If a message passing system were used, it would be necessary to explain message passing.
These can be complicated to explain, and come with computational overhead.
In contrast, class coupling is fast and easy to explain.
